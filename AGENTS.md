# Agent instructions

## Keep the readme in sync

Whenever you make a change, consider whether [readme.md](readme.md) needs an update. Not
every change does — internal refactors, bug fixes with no visible effect, and the like
usually don't. But anything that changes how the app works, what a user can do, or how
they do it should be reflected in the readme. This includes new features, changed or
removed behavior, new keyboard/mouse interactions, and changes to setup or deployment.
